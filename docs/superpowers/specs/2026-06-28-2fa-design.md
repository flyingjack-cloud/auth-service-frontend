# 2FA Support Design

**Date:** 2026-06-28  
**Scope:** auth-portal — login 2FA challenge flow + security page 2FA management

---

## Overview

Add TOTP-based two-factor authentication to two places:

1. **Login flow** — when the backend returns 202, route the user through a TOTP verification step before establishing a session.
2. **Security page** — let users enable, confirm, and disable 2FA from the account security settings.

All 2FA API endpoints are already documented in `API.md` (section I). No backend changes required.

## Services

### TwoFaService

Stateless HTTP wrapper. All observables resolve to unwrapped `data` (handled by the existing `apiInterceptor`).

```typescript
verify(req: TwoFaVerifyRequest): Observable<User>         // POST /account/2fa/verify
setup(): Observable<TwoFaSetupResponse>                   // POST /account/2fa/setup
confirm(req: { code: string }): Observable<null>          // POST /account/2fa/confirm
disable(req: TwoFaDisableRequest): Observable<null>       // DELETE /account/2fa
```

### TwoFaStatusService

Reads and writes a single `localStorage` key (`2fa_enabled`). Exposes a readonly Signal so components react automatically.

```typescript
readonly enabled: Signal<boolean>   // true iff localStorage key === 'true'
setEnabled(v: boolean): void        // updates localStorage + signal
clearStatus(): void                 // removes key, sets signal to false
```

`AuthService.logout()` calls `TwoFaStatusService.clearStatus()` so the flag resets on session end.

## Login 2FA Flow

### AuthService.login() change

Switch from the default `observe: 'body'` to `observe: 'response'` so the HTTP status code is visible:

- Status **200** → existing behavior: tap sets `currentUser`, emit user to subscriber
- Status **202** → body is `PendingTokenResponse`; do NOT set `currentUser`; emit a typed result that the component can distinguish

The service returns `Observable<{ kind: 'ok'; user: User } | { kind: '2fa'; pendingToken: string }>` (or equivalent discriminated union) so the component doesn't need to inspect HTTP internals.

### LoginComponent change

In `onSubmit()`, after a `kind: '2fa'` result:
1. Read `redirect_uri` from the current query params (preserve it)
2. Navigate to `/2fa-verify` with Router navigation extras `state: { pendingToken, redirectUri }`

### /2fa-verify component

- On init: reads `history.state.pendingToken`. If absent (e.g. hard refresh), redirects immediately to `/login`.
- Renders an `auth-card` (same pattern as login/register) with:
  - Lock icon, title, subtitle
  - Single 6-digit number input (maxlength 6, inputmode numeric)
  - Submit button with loading spinner
  - "返回登录" back link
- On submit: calls `TwoFaService.verify({ pendingToken, code })`
  - Success → `AuthService.setCurrentUser(user)`, navigate to `redirectUri ?? '/account'`
  - `error.2fa.invalid-token` → redirect to `/login` (session expired)
  - `error.2fa.invalid-code` → inline error, keep form

## Security Page 2FA Card

Added below the change-password card. Renders differently based on `TwoFaStatusService.enabled`.

### State: disabled

```
[ shield icon ]  双因素验证 (2FA)
                 使用 Authenticator App 为账号增加一层保护
                                              [ 启用 2FA ]
```

Clicking "启用 2FA":
1. Calls `TwoFaService.setup()` → receives `otpAuthUri`
2. Inline expansion appears:
   - QR code rendered via `angularx-qrcode` from `otpAuthUri`
   - 6-digit confirm input
   - Submit button
3. Submit calls `TwoFaService.confirm({ code })`
   - Success → `TwoFaStatusService.setEnabled(true)`, show success notice, collapse form

### State: enabled

```
[ shield icon ]  双因素验证 (2FA)      [ 已启用 badge ]
                 已通过 Authenticator App 保护
                                              [ 停用 2FA ]
```

Clicking "停用 2FA":
- Inline form expands: current password + TOTP code input + confirm button
- Submit calls `TwoFaService.disable({ password, code })`
  - Success → `TwoFaStatusService.setEnabled(false)`, collapse form

Error messages are shown inline (reuse `sh-error-alert`). No dialogs.

## Dependency

Add `angularx-qrcode` to `package.json` for client-side QR code rendering. The `otpAuthUri` (containing the TOTP secret) never leaves the browser.

## Error Handling Summary

| Error ID | Location | Behavior |
|---|---|---|
| `error.2fa.invalid-token` | `/2fa-verify` | Redirect to `/login` |
| `error.2fa.invalid-code` | `/2fa-verify` | Inline error, retry |
| `error.2fa.invalid-code` | Security setup confirm | Inline error |
| `error.2fa.invalid-code` | Security disable | Inline error |
| `error.user.wrong-password` | Security disable | Inline error |
| Any other | All | Generic inline error |
